🔗
noUpdates: boolean

Pass true to disable incoming updates.<br/> When updates are disabled the client can't receive messages or other updates.<br/> Useful for batch programs that don't need to deal with updates.

🔗
takeout: boolean

Pass true to let the client use a takeout session instead of a normal one, implies noUpdates : true.<br/> Useful for exporting Telegram data. Methods invoked inside a takeout session (such as get_chat_history,download_media, ...) are less prone to throw FloodWait exceptions.<br/> Only available for users, bots will ignore this parameter.

🔗
tcp: number

TCP Modes.<br/> Fill with index of tcp modes if you want to switch from one tcp to another. <br/> Default is 1 (TCPAbridge)

🔗
local: boolean

Only for browser platform! Set false when you deployed your app offside the local machine. it wil use ws:// in local machine and wss:// in deployment.

🔗
maxConcurrentTransmissions: number

Set the maximum amount of concurrent transmissions (uploads & downloads). A value that is too high may result in network related issues. Defaults to 1.
